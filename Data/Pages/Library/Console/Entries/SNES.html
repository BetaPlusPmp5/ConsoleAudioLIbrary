Sony SPC700 Sound Processing Unit

The SPC700 is the audio processing unit used in the Super Nintendo Entertainment System (SNES). This chip was revolutionary for its time, allowing games to feature rich, layered, and sample-based music far beyond the bleeps and bloops of earlier consoles. It worked alongside a digital signal processor (DSP) and had its own dedicated 64KB of RAM for audio data.

Unlike the NES, the SPC700 is not monophonic. In fact, it supports 8 channels of audio, each of which can play independent digital samples simultaneously. This means each channel can play different notes and instruments at the same time — one channel can play a bassline, another drums, another voice, etc.

The chip uses ADPCM (Adaptive Differential Pulse Code Modulation) to compress sound samples so they can fit into memory, and the noise channel is actually a feature of the DSP, which uses noise generation algorithms to simulate sounds like snares and explosions by randomly fluctuating audio waveforms. This gives SNES games their distinctive percussion and ambient effects.

A special format called SPC files lets users rip music from SNES games. These files contain the full state of the SPC700 (its RAM, registers, etc.), and when played back through emulators or music players, they sound exactly like the original game music — because they essentially are.

  • Manufacturer: Sony
  • Release Year: 1990

Architecture Overview

The SPC700 is a dedicated 8-bit microprocessor that controls sound playback on the SNES. It works in conjunction with the DSP (Digital Signal Processor), which handles actual audio output. Music and effects are stored as compressed samples in RAM, then mixed and played through the 8 available channels. The CPU sends instructions and data to the SPC700 through memory writes, allowing music to change dynamically based on gameplay.

Below is a breakdown of how the SNES makes sound using the SPC700. It has 8 channels, and each one can be assigned any sample sound. These channels are polyphonic — meaning multiple notes can be played at the same time across channels.

  • Channel: The lane used to play one sound or instrument. Each channel can have its own volume, pitch, and effects.
  • Sample: A short, recorded sound like a piano note, drum hit, or voice clip.
  • Pitch Control: Change how high or low the note sounds. Almost all samples can be adjusted to different pitches.
  • Envelope: Controls the shape of a sound — how fast it starts, how long it lasts, and how it fades away.
  • Echo: A repeating version of the sound that trails off, like in a cave or tunnel. SNES used real-time echo effects.

Composers had access to tools that let them program how each channel behaved — layering instruments, creating sweeping effects, or even simulating surround sound using clever panning and echo.

Technical Summary

  • Total Channels: 8
  • Polyphonic: Yes — each channel plays one sample at a time, but multiple channels combine for full music
  • Sample Rate: ~32kHz
  • Output: Stereo
  • DSP Features: Echo, filtering, pitch modulation, noise

Limitations

  • Limited RAM (64 KB) for all samples and code
  • No real-time streaming — samples must be preloaded
  • Noise generation limited to DSP features
  • No MIDI compatibility by default

Modern Development Tools

AddmusicK

A tool designed for inserting custom music into SNES ROM hacks, especially Super Mario World. Supports the full SPC engine and samples.

SNESGSS

A graphical music sequencer for creating music for the SNES using the SPC700. It mimics the SNES music engine and supports sample importing and echo effects.

SPCPlay

SPCPlay is a lightweight SPC player that lets users listen to .spc files from real SNES games. It emulates the SPC700 perfectly, letting composers test their work or enjoy SNES music outside of the console.
